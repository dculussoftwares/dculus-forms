Feature 2 AI validation suggestions (proposeValidation tool) Backend + Frontend

AI proposes, user accepts or dismisses per field

Here are suggested validation rules based on your field labels:
"Phone number" (text)
maxLength: 15 · pattern: digits only
"Age" (number)
min: 0 · max: 120
Backend: New read-only proposeValidation tool — takes an array of { fieldId, label, type }, returns { type: 'PROPOSE_VALIDATION', suggestions: [...] }. Execute function is pure JS (no DB call — uses schema passed at agent creation time). System prompt updated: "Use proposeValidation when asked to suggest or review validation rules, never call updateField directly for validation without user confirmation."

Frontend: New ValidationSuggestionCard.tsx tool-part component. applyAIOp does NOT auto-apply PROPOSE_VALIDATION — it sets pendingValidationSuggestions in a new Zustand slice. Accept dispatches UPDATE_FIELD per field; Skip removes it from pending.
Feature 3 Bulk field operations (bulkUpdateFields tool) Backend + Frontend · replaces N updateField calls

Before — "Make all fields required"

Step 1: listFields → 8 field IDs
Step 2: updateField(f1)
Step 3: updateField(f2)
Step 4: updateField(f3)
... 5 more steps
Total: 9 tool calls

After — with bulkUpdateFields

Step 1: listFields → 8 field IDs
Step 2: bulkUpdateFields(
  fieldIds: [f1..f8],
  updates: { required: true }
)
Total: 2 tool calls
Backend: New bulkUpdateFields tool — takes { fieldIds: string[], updates: UpdatesSchema } (same updates schema as updateField), returns { type: 'BULK_UPDATE_FIELDS', fieldIds, updates }. System prompt updated: "Use bulkUpdateFields instead of multiple updateField calls when applying the same change to 3 or more fields."

Frontend: Handle BULK_UPDATE_FIELDS in applyAIOp.ts — iterate fieldIds, apply UPDATE_FIELD for each within the existing Y.js batch. ChangeSummaryCard renders it as "~ Updated N fields (required)".
Feature 4 "Remix this form" command System prompt + chip · no new tool needed

AI orchestrates existing tools to transform the form's purpose

Make this a job application form instead
I'll transform the current contact form into a job application form. Let me restructure the fields:
− Removed "Message" (textarea)
− Removed "Subject" (text)
+ Added "Position applied for" (select)
+ Added "Years of experience" (number)
+ Added "Cover letter" (textarea)
+ Added "Resume / CV" (file)
~ Updated layout title → "Job Application"
No new tool: Remix orchestrates listFields → removeField → addField → updateLayout using existing tools. The change summary card (Phase 2) makes the diff visible. Per-message undo (Phase 2) lets the user revert instantly.

System prompt addition: "When asked to 'remix', 'transform', or 'convert' the form for a different purpose: (1) call listFields to understand existing structure, (2) remove fields that don't fit the new purpose, (3) add fields that belong, (4) update the layout title and CTA. Preserve fields that work for both purposes."

New context chip (useAIChips): When the form has > 2 fields, add "✦ Remix this form" chip that sends the prompt above pre-filled with a textarea asking for the new purpose.
