Design: Responsibility Map

Section 1 of 4 — Who owns what. Each box has one job.

Full System Flow
Frontend (form-app)
AIEditDrawer.tsx
Pure UI only. No business logic.
Renders messages, undo button, streaming chips.
Passes currentPageId down to hook.
useAIChat hook
Owns conversation list + CRUD (GraphQL).
Wires useAIStream + useUndoStack.
Exposes clean API to drawer.
useAIStream hook ← NEW
fetch() POST → reads NDJSON stream.
On text: update bubble.
On operation: call applyAIOp().
On done: refetch conversation.
applyAIOp() ← NEW
Pure function. Single place for all
store mutations triggered by AI.
Handles position, multi-page, options.
Fully unit-testable.
useUndoStack hook ← NEW
Snapshots Zustand state before
first op in each AI response.
undo() restores snapshot.
Cleared on next user message.
POST /api/ai/chat
↑ stream NDJSON
Backend (Express)
POST /api/ai/chat ← NEW ROUTE
Auth check. Save user message.
Call buildChatStream().
Pipe NDJSON chunks to response.
On finish: save assistant msg + tokens.
aiChatService.ts (simplified)
buildChatStream(conversationId, userId)
No currentFormState arg — AI fetches
what it needs via tools.
System prompt: short, no form state baked in.
aiFormEditTools.ts (rewritten)
listFields(pageId?) → live from Prisma
getField(fieldId) → full details
addField / updateField / removeField
reorderFields / updateLayout
(no more dummy inspectForm)
GraphQL (unchanged)
Conversation CRUD stays GraphQL.
Subscription removed entirely.
No more AI_CHAT_STREAM in schema.
Key rule: The AI never receives currentFormState from the client. It calls listFields() and getField() to read live data from the DB. The frontend only sends { conversationId, organizationId, currentPageId }.
